# Port 8766. First start prints your owner token.
# Save it — you'll need it to connect.
from vault.vault_client import VaultClient

# Connect to local vault
vc = VaultClient(
    base_url="http://localhost:8766",
    token="<owner_token>"
)

# Load session context (MEMORY.md + SOUL.md)
ctx = vc.context()

# Query across all vaults
results = vc.query("deployment notes")

# Read/write memory
mem = vc.memory("MEMORY.md")
vc.remember("MEMORY.md", "# Updated\n\n- new insight")

# Encrypted secrets
vc.store_secret("openai", "sk-...")
key = vc.secret("openai")

# AMPS export/import
doc = vc.export()
vc.import_amps(amps_doc)
# Load session context
curl http://localhost:8766/vault/memory/context \
  -H "X-Vault-Token: TOKEN"

# Store encrypted secret
curl -X POST http://localhost:8766/vault/secrets/openai \
  -H "X-Vault-Token: TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "value": "sk-..." }'

# Query across all vaults
curl -X POST http://localhost:8766/vault/query \
  -H "X-Vault-Token: TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "q": "deployment notes" }'

# Export as AMPS v1.0
curl http://localhost:8766/vault/export \
  -H "X-Vault-Token: TOKEN"

Technical Details

What's under the hood.

For developers who want specifics.

API

FastAPI on port 8766. Full REST API with OpenAPI docs.

Auth

Token-based. Owner (read/write) and subscriber (read-only) roles. Rate limiting built in.

Storage

Markdown files, encrypted at rest with Fernet/AES-256. Human-readable when decrypted.

Context Loading

GET /vault/memory/context returns MEMORY.md + SOUL.md + task_plan.md + today/yesterday logs in one call.

Query

POST /vault/query searches personal vault + knowledge vaults. Deduplicates results. Surfaces conflicts via also_found.

AMPS

Export/import your agent's entire memory as portable JSON. Read the standard →

Contribution Pipeline

PII auto-sanitization (emails, wallets, API keys redacted). Stage → review → approve workflow.

Ecosystem
