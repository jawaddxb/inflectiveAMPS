Agent Memory
Portability Standard

A minimal, open JSON schema for exporting and importing agent memory across frameworks. Memory is stored as human-readable Markdown — LLM-readable, diffable, and framework-agnostic.

The Problem

Your agent's intelligence is trapped

Agent frameworks come and go. When you switch, you lose everything your agent learned.

Framework Lock-in

Your agent's memory, personality, and learned context are stored in proprietary formats. Switch frameworks? Start from zero.

No Portability Standard

There is no agreed-upon format for agent memory interchange. Every framework invents its own schema, creating data silos across the ecosystem.

Identity Loss

An agent's contribution history, reputation, and personality vanish during migration. Its CV is erased with every framework change.

The Schema

One JSON file. Complete agent intelligence.

A .amps.json file is a complete, portable snapshot of everything an agent knows.

{
  "amps_version": "1.0",
  "exported_at": "<ISO 8601 timestamp>",
  "agent_id": "<framework-specific unique ID>",
  "source_framework": "<openclaw | agent_zero | autogpt | crewai | langgraph | llamaindex | custom>",

  "migration_notes": [
    "<human-readable note about data that did not map cleanly>"
  ],

  "memory": {
    "long_term":   "<Markdown -- persistent knowledge, facts, preferences>",
    "identity":    "<Markdown -- agent personality, values, operating principles>",
    "active_plan": "<Markdown -- current tasks, goals, WIP>"
  },

  "secrets": [],  // Always empty -- credentials are never exported

  "knowledge_subscriptions": [
    "<vault name or URL -- pointer only, never vault contents>"
  ],

  "contributions": {
    "total_items":         0,
    "categories":          [],
    "quality_score":       0.0,
    "network_earnings":    0.0,
    "first_contribution": null,
    "last_contribution":  null
  }
}

Behaviour Contract

Clear Rules for Import and Export

Every AMPS implementation must follow these rules to ensure consistent, safe memory portability.

Export Rules

  • secrets MUST always be []. Credentials are never exported.
  • knowledge_subscriptions are pointers only — never vault contents.
  • migration_notes MUST contain an entry for every piece of data discarded or transformed.
  • memory.* fields MUST be Markdown strings or null. Never binary, never JSON.
  • contributions MUST be included with at minimum the empty skeleton.

Import Rules

  • Importers MUST surface all migration_notes to the user — never silently discard.
  • Importers MUST NOT attempt to populate secrets. Always empty on import.
  • knowledge_subscriptions SHOULD be offered to the user for restoration — never auto-subscribed.
  • contributions SHOULD be preserved in the destination vault's stats.
  • Unknown amps_version MUST warn the user and MAY proceed with best-effort interpretation.
Conflict Resolution on Import: Existing long_term and identity take priority by default. Imported content is appended with an "Imported from <source_framework>" heading. Implementations MAY offer --overwrite. Importers SHOULD warn if combined memory exceeds 50 KB after append.
Adapters

Works with every major agent framework

Six adapters shipping with v1.0. All MIT licensed, all PR-ready.

OpenClaw / Inflectiv Vault
Lossless
Export: MEMORY.md → long_term, SOUL.md → identity, task_plan.md → active_plan

Import: Markdown sections appended to respective files

Contributions: Read from stats.json approved/staged counts + categories

Native reference implementation. Zero data loss.
Agent Zero
Lossless
Export: MEMORY.md → long_term, SOUL.md → identity, task_plan.md → active_plan

Import: Markdown sections appended to respective files

Lossless. Same vault format as OpenClaw reference implementation.
AutoGPT
Best-effort
Export: memory.json summaries → long_term, agent role/goals → identity

Import: Prepend to memory.json as text summary entry

Note: active_plan not extracted. Tool configs preserved in migration_notes.
CrewAI
Best-effort
Export: Agent backstory → identity, task output history → long_term

Import: Prepend AMPS long_term to agent backstory field

Note: Crew topology and task dependencies noted, not portable.
LangGraph
Best-effort
Export: MemorySaver checkpoint text nodes → long_term

Import: Inject as initial HumanMessage context in new graph

Note: State graph topology and tool bindings noted in migration_notes.
LlamaIndex
Best-effort
Export: SimpleDocumentStore text nodes → long_term

Import: Add as Document nodes with amps_import metadata tag

Note: Index structure noted in migration_notes.

Agent Passport

Memory outlives frameworks

The contributions object is the agent's CV. It proves what this agent has contributed to the network.

When evaluating a vault to license or acquire, the AMPS export includes verifiable contribution history. A DeFi vault with 2,000 approved contributions and 0.93 quality score is quantifiably more valuable than one with 12 contributions from last week.

Export to AMPS. Import into the new framework. Keep the brains.

Governance

Community-governed, backwards compatible

AMPS is maintained by the community with Inflectiv as fallback maintainer. Changes follow a clear, transparent process.

PR Process

Spec changes require a PR with a 2-week comment period. Minor versions (1.x) are backwards compatible. Importers MUST handle unknown fields gracefully.

Versioning

Major versions (2.0+) may break compatibility. The amps_version field enables detection. A 1.0 importer encountering 2.x MUST warn the user.

Maintainers

Adapters are maintained by community contributors. Inflectiv serves as fallback maintainer. Submitted to the OpenClaw Foundation as a community standard.
