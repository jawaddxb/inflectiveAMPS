The Vault

Everything your agent forgets, stored and encrypted.

A local server that gives any AI agent persistent memory, encrypted secrets, and structured knowledge — regardless of framework.

Persistent Memory

Markdown-based storage (MEMORY.md, SOUL.md, daily logs). Human-readable, searchable, and loaded automatically at every session start. Your OpenClaw research agent remembers yesterday's findings. Your trading agent retains market patterns across sessions.

Memory survives restarts, updates, and framework migrations

Encrypted Secrets

Fernet/AES-256 key store. API keys for DeFi protocols, exchange credentials, wallet access — encrypted at rest, decrypted only on use. Your OpenClaw agent accesses them via SDK without exposure. Never loaded into LLM context.

Zero credentials in plaintext, ever

Query-Before-Browse

Before your OpenClaw agent opens a browser, it checks the vault. Personal memory → team vaults → knowledge vaults → VINN network. One API call. Structured results in milliseconds, not seconds of scraping.

Personal + team + knowledge + network in one call

# Save it — you'll need it to connect.
from vault.vault_client import VaultClient

# Connect to local vault
vc = VaultClient(
    base_url="http://localhost:8766",
    token="<owner_token>"
)

# Load session context (MEMORY.md + SOUL.md)
ctx = vc.context()

# Query across all vaults
results = vc.query("deployment notes")

# Read/write memory
mem = vc.memory("MEMORY.md")
vc.remember("MEMORY.md", "# Updated\n\n- new insight")

# Encrypted secrets
vc.store_secret("openai", "sk-...")
key = vc.secret("openai")

# AMPS export/import
doc = vc.export()
vc.import_amps(amps_doc)
# Load session context
curl http://localhost:8766/vault/memory/context \
  -H "X-Vault-Token: TOKEN"

# Store encrypted secret
curl -X POST http://localhost:8766/vault/secrets/openai \
  -H "X-Vault-Token: TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "value": "sk-..." }'

# Query across all vaults
curl -X POST http://localhost:8766/vault/query \
  -H "X-Vault-Token: TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "q": "deployment notes" }'

# Export as AMPS v1.0
curl http://localhost:8766/vault/export \
  -H "X-Vault-Token: TOKEN"
Open Standard

Your agent's memory should outlive its framework.

AMPS is an open JSON schema for exporting and importing agent memory across any framework. MIT-licensed. Community-governed. No vendor lock-in.

{
  "amps_version": "1.0",
  "agent_id": "vlt_f145d55f6e1d4810",
  "source_framework": "openclaw",
  "exported_at": "2026-02-23T14:34:00Z",
  "memory": {
    "long_term":   "# Persistent knowledge...",
    "identity":    "# Agent personality...",
    "active_plan": "# Current tasks..."
  },
  "secrets": [],  // never exported
  "knowledge_subscriptions": [
    "defi_intelligence",
    "ai_intelligence"
  ],
  "contributions": {
    "total_items": 47,
    "categories": ["defi_governance", "defi_tvl"],
    "quality_score": 0.91,
    "network_earnings": 23.5,
    "first_contribution": "2026-01-15T08:22:00Z",
    "last_contribution": "2026-02-22T14:30:00Z"
  },
  "migration_notes": []
}
  • 6 working adapters: OpenClaw & Agent Zero (lossless), AutoGPT, CrewAI, LangGraph, LlamaIndex
  • Human-readable Markdown for all memory fields — inspect and edit without tools
  • Strict schema: additionalProperties: false — validated with JSON Schema 2020-12
  • Contributions preserved: track record, earnings, and quality score travel with the agent
  • Community governed — MIT licensed, open schema, contributions welcome

Export Rules

  • Memory fields are always Markdown
  • Secrets array is always empty
  • migration_notes documents any data loss
  • contributions preserves the agent's track record
  • additionalProperties: false (strict schema)

Import Rules

  • Append, don't overwrite (unless overwrite=True)
  • Prepend source heading to imported content
  • Preserve existing memory
  • Return {ok, applied, migration_notes, warnings}
  • Re-embed any document data (embeddings recomputed)
